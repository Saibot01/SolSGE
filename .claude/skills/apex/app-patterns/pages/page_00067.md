## Patrones clave de este módulo

### Precio por categoría de cliente
El IG de detalle de venta usa la tabla `PRECIO_POR_CATEGORIA` para filtrar productos disponibles según la categoría del cliente:

```sql
-- Columna PRODUCTO en IG — LOV inline
select pro.nombre, pro.id_producto
from productos pro, precio_por_categoria cat
where pro.id_producto = cat.id_producto
  and cat.CATEGORIA_CLIENTE = :P67_TIP_CLIENTE
```

### recalculaImporte() — versión ventas con IVA

Más compleja que la de compras: desglosa IVA 5% e IVA 10% leyendo columnas TOTAL, IVA_CALCULADO y PORCENTAJE del IG.

```javascript
function recalculaImporte() {
    var model = apex.region("Detalle_V").widget().interactiveGrid("getCurrentView").model;
    // Lee col TOTAL, IVA_CALCULADO, PORCENTAJE
    // Acumula total_iva_5, total_iva_10, total_general
    apex.item('P67_TOTAL_IVA_5').setValue(Math.round(total_iva_5));
    apex.item('P67_TOTAL_IVA_10').setValue(Math.round(total_iva_10));
    apex.item('P67_TOTAL_IVA').setValue(Math.round(total_iva_5 + total_iva_10));
    apex.item('P67_TOTAL_MONEDA_LOCAL').setValue(Math.round(total_general));
}
```

## Notas de negocio

- El proceso de ventas trabaja directamente sobre COMPROBANTES (factura de venta)
- El detalle es DETALLE_ORDEN donde el join va por ORDENES_VENTA
- Los precios varían por CATEGORIA_CLIENTE: A, B, C (tabla PRECIO_POR_CATEGORIA)
- El IVA se calcula según TIPO_IVA asociado al producto
- La región es `p_ajax_items_to_submit=>'P67_ID_ORDEN_VENTA'` para refrescar el IG al cambiar orden
