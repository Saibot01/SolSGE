### CHK_LIMITE_MONTO_MENSUAL (PL/SQL validation)

```sql
DECLARE
  l_limite     NUMBER := FN_GET_LIMITE_OC_MENSUAL(TO_CHAR(SYSDATE,'MM'), TO_CHAR(SYSDATE,'YYYY'));
  l_acumulado  NUMBER;
BEGIN
  IF l_limite IS NULL THEN RETURN; END IF;
  SELECT NVL(SUM(TOTAL_ORDEN),0) INTO l_acumulado
  FROM ORDENES_COMPRA
  WHERE TRUNC(FECHA_ORDEN,'MM') = TRUNC(SYSDATE,'MM')
    AND ESTADO != 'A'
    AND ID_ORDEN_COMPRA != NVL(:P72_ID_ORDEN_COMPRA,-1);
  IF (l_acumulado + :P72_TOTAL_ORDEN) > l_limite THEN
    APEX_ERROR.ADD_ERROR(P_MESSAGE=>'Límite mensual OC superado...', P_DISPLAY_LOCATION=>APEX_ERROR.C_INLINE_IN_NOTIFICATION);
  END IF;
END;
```

### IG DML (custom PL/SQL)

```sql
begin
  case :APEX$ROW_STATUS
    WHEN 'C' THEN INSERT INTO DETALLE_ORDEN_COMPRA(ID_ORDEN_COMPRA,ID_PRODUCTO,CANTIDAD,PRECIO_UNITARIO,TOTAL_DETALLE)
                  VALUES (:P72_ID_ORDEN_COMPRA,:ID_PRODUCTO,:CANTIDAD,:PRECIO_UNITARIO,:TOTAL_DETALLE);
    WHEN 'U' THEN UPDATE DETALLE_ORDEN_COMPRA SET ID_PRODUCTO=:ID_PRODUCTO, CANTIDAD=:CANTIDAD,
                  PRECIO_UNITARIO=:PRECIO_UNITARIO, TOTAL_DETALLE=:TOTAL_DETALLE WHERE ID_ORDEN_COMPRA=:P72_ID_ORDEN_COMPRA;
    WHEN 'D' THEN DELETE DETALLE_ORDEN_COMPRA WHERE ID_DETALLE_OC=:ID_DETALLE_OC;
  end case;
end;
```

## JavaScript en página

```javascript
function recalculaImporte() {
    var model = apex.region("Detalle_Orden_Compra").widget().interactiveGrid("getCurrentView").model;
    var col = model.getFieldKey("TOTAL_DETALLE");
    var n_total = 0;
    model.forEach(function(row) {
        var v = row[col];
        if (v !== null && v !== undefined) {
            var clean = v.toString().replace(/[^0-9,.-]/g,'').replace(/\./g,'').replace(',','.');
            var n = parseFloat(clean);
            if (!isNaN(n)) n_total += n;
        }
    });
    apex.item('P72_TOTAL_ORDEN').setValue(Math.round(n_total));
}
```

## Patrones clave observados

1. **Estado oculto en borrador:** P72_ESTADO='B' se guarda como default; el proceso de aprobación lo cambia
2. **Empleado auto-resuelto:** P72_ID_EMPLEADO se carga vía SQL por APP_USER, nunca se edita
3. **Precio desde vista vigente:** `V_PRODUCTO_PROVEEDOR_VIGENTE` filtra por VIGENCIA='VIGENTE' + ID_PERSONA
4. **LOV multicolumna:** PRODUCTOS.PROVEEDOR.PRECIO muestra Producto, Proveedor, Precio, PrecioMin, Diferencia — útil para comparar precios al seleccionar
5. **Validación de negocio en proceso:** CHK_LIMITE usa `FN_GET_LIMITE_OC_MENSUAL` del módulo de Parámetros
